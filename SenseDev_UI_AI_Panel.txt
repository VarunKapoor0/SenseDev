SenseDev AI Panel Redesign Plan
Right-Panel Tab System + AI Assistant UI Specification
Purpose

The AI assistant currently lives in a small lower-left panel. Its position and visual design reduce usability and visibility.
This document proposes a new design:

AI panel moved to the right side, inside a new tabbed right-panel container

Tabs: Details and AI

Clean, modern, elegant styling suitable for a professional devtool

A proper full-height panel for AI messages & interactions

This significantly enhances usability, consistency, and visual polish.

1. Right-Panel Architectural Changes

The existing right panel (currently showing Details) becomes a shared container with tabs.

1.1 New Panel Layout
+--------------------------------------------------------------+
| [ Details ]   [ AI ]                                        |
+--------------------------------------------------------------+
|                                                              |
|   (active panel content here: DetailsView OR AIChatView)     |
|                                                              |
+--------------------------------------------------------------+

1.2 Requirements

The entire panel is height-matched to the graph area

Tabs appear at the top of the right panel, not at the top of the window

Switching tabs must not resize the graph or sidebar

The right panel must support two views:

DetailsView(selectedNode)

AIChatView()

1.3 Interaction Behavior

Clicking Details shows node metadata

Clicking AI shows the AI chat interface

Last-used tab persists per session

When switching from Details → AI:

The AI tab should show context for the currently selected node (if any)

Smooth transitions (AnimatedContent or crossfade)

2. Tab Bar Visual Specification

Tabs must feel modern—no square blocks, no thick borders.

2.1 Tab Bar Container

Height: 40–44px

Background: #1A1A1A

Bottom border: 1px solid #2A2A2A

Horizontal padding: 12px

Space between tabs: 8–12px

2.2 Tab Appearance
State	Style
Default	- Text: #A0A0A0
- Background: transparent
- Padding: 10px vertical, 14–18px horizontal
- Border-radius: 6–8px
- Hover: background #242424
Selected	- Text: #E8E8E8 (strong contrast)
- Background: #262626
- Border-radius: 8px
- Bottom accent bar: 2px accent (#5FD3FF or #8E82FF)
- Slight shadow for depth
Hover	- Smooth color fade (150ms)
- Slight brightness ↑
2.3 Interaction Design

Switching tabs triggers a crossfade animation

Tab indicator moves smoothly (not instant switch)

Clicking on a tab does not shift layout; content should swap inside the container

3. AI Panel UI Specification (Inside the AI Tab)

The AI panel must look like a first-class feature, not an afterthought.

It should resemble:

Cursor AI chat

Sourcegraph Cody

VSCode Copilot panel

3.1 Layout Structure
+---------------------------------------------------+
|  AI Tab Header (optional model + context badges)  |
+---------------------------------------------------+
|  Scrollable message area (full height - input)    |
|                                                   |
|                                                   |
|---------------------------------------------------|
|  Input bar (rounded, fixed height)                |
+---------------------------------------------------+

3.2 Message Area
Requirements:

Vertical scrolling

Automatic scroll-to-bottom on new message

Different styling for user vs AI messages

AI message bubble:

Background: #222222

Border-radius: 8px

Padding: 12–14px

Markdown support

Monospace fonts for code blocks

Syntax highlighting (later iteration)

User message bubble:

Background: #2D2D2D

Border-radius: 8px

Padding: 12–14px

Alignment: right-aligned

3.3 Message Metadata

Optional (high polish):

Timestamp dimmed

Node context badge:
Context: MainActivity.kt

Status indicator:
gemini-2.0-flash-exp ✓

4. Input Bar Specification

The input bar is critical. It must feel premium.

4.1 Visual Design

Height: 48–54px

Rounded corners: 10–12px

Background: #1F1F1F

Border: 1px solid #2A2A2A

Shadow (subtle):
0 1px 3px rgba(0,0,0,0.3)

Placeholder text:
"Ask about this codebase…", color #6F6F6F

4.2 Send Button

Icon only (paper airplane or arrow)

Accent-colored (#5FD3FF)

Hover glow effect

Positioned at right edge inside input bar

4.3 Behavior

Press Enter to send

Shift+Enter = newline

Disabled state if no model is selected

5. AI Panel Functional Behavior Specification
5.1 Context Handling

When the user has a selected node:

AI panel displays "Context: MainActivity" or similar

Messages sent should include node metadata unless disabled

When no node is selected:

AI panel defaults to “general codebase questions”

5.2 AI Provider Display

At the top-right area of the AI panel:

Model: gemini-2.0-flash-exp 
(full file access)

5.3 Rate Limit / Errors

Show graceful, styled error messages:

Soft red background: #3A1F1F

Border: #6A3333

Text: #E0A0A0

Padding: 8px

Rounded corners

6. Details Panel Adjustments (to match AI style)

The Details panel does not need major changes but must visually match the AI panel so switching tabs feels smooth.

Key refinements:

Increase title size & weight

Add accent underline or subtle divider

Add consistent badges for node type

Replace bullet list with cleaner list rows:

Icon → Label → Relation type

Example list entry:
↳ PreferencesKeys     WRITES_STATE

Colors matched with AI tab so transitions feel cohesive.
7. Right-Panel Container Behavior
7.1 Shared Container

Same padding for both tabs

Same background color (#1A1A1A)

Uniform typography

Uniform spacing principles

7.2 Sizing

The panel width remains constant (good for muscle memory)

Min width: 300px

Preferred width: 340–380px

7.3 Panel Resizer (Optional)

Allow dragging the left edge of the right panel to resize it.

8. Animation Specification

Animations must be subtle and fast.

8.1 Tab switching animation

Crossfade: 120ms

Or slide-left/slide-right: 150ms

8.2 Hover interaction for tabs

Background fades in/out: 120ms

Text color easing: 100ms

8.3 Message arrival animation

Fade + slight upward slide: 120ms
